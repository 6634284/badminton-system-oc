# 06 · 后台管理系统设计

> 工程：`admin-web/`
> 技术栈：**React 18 + TypeScript 5 + Vite 5 + Ant Design 5 + Ant Design Pro Components**
> 双形态：
> - **平台后台（Platform Admin）**：平台运营使用，管理租户入驻、套餐、平台佣金、风控。
> - **租户后台（Tenant Admin）**：俱乐部主理人 / 员工使用，管理自家球馆、活动、会员、商城、财务。
>
> 二者复用同一份代码 + 路由权限，按登录账号类型动态生成菜单。

function AuthGuard({ permission, children }: Props) {
  const { hasPermission } = usePermission();
  if (!hasPermission(permission)) return <Result status="403" />;
  return children;
}

### 3.3 鉴权与多租户
```ts
// stores/auth.ts (Zustand)
interface AuthState {
  user: AuthUser | null;
  tenantId: number | null;        // 平台账号可切换
  permissions: string[];
  menuTree: MenuItem[];
  login: (req) => Promise<void>;
  logout: () => void;

```
- 登录成功后存 token（httpOnly cookie + sessionStorage 双备份）。
- axios 拦截器：401 → 自动 refresh，连续失败 → 跳登录。
- 平台账号可通过顶部「切换租户」下拉切换调试视角。

### 3.4 服务端状态 — TanStack Query
- 所有列表 / 详情走 `useQuery`，写操作走 `useMutation`，自动失效相关查询缓存。
- 工厂模式封装：
```ts
// services/modules/activities.ts
export const activityKeys = {
  list: (params) => ['activities', params] as const,
  detail: (id) => ['activities', id] as const,
};

export function useActivityList(params: ActivityListParams) {
  return useQuery({
    queryKey: activityKeys.list(params),
    queryFn: () => api.activities.list(params),
    staleTime: 30_000,
  });
}

export function useCreateActivity() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: api.activities.create,
    onSuccess: () => qc.invalidateQueries({ queryKey: ['activities'] }),
  });
}
```

### 3.5 主题与暗色模式
- AntD v5 token 配置统一：主色、圆角、字体；
- 通过 `useThemeToggle` 切换 light / dark；
- 后台色板偏「商务深蓝」，重点动作色「绿/橙」。

## 4. 模块详述

### 4.1 经营总览 Dashboard
- 顶部 4 张关键卡片（今日 GMV / 订单数 / 活跃会员 / 报名场次）。
- 主图：近 30 天 GMV / 订单趋势（双 Y 轴）。
- 副图：会员等级分布、商品销售排行。
- 实时通知 Drawer：新订单、取消、退款。

### 4.2 球馆 / 场地 / 排期
- **球馆列表**：地址、营业时间、场地数、状态。
- **场地管理**：批量新增、编号重排、维护（可批量勾选时段标记 maintenance）。
- **排期管理**：
  - 横向 Timeline 视图：时间在 X 轴，场地在 Y 轴；色块代表 available/held/booked/maintenance。
  - 鼠标拖拽框选 → 批量改价、维护、取消维护。
  - 切日 / 切球馆。
  - 关键组件：`<CourtScheduleGrid>`，使用 `react-window` 虚拟滚动应付大球馆 30 块场地 × 48 时段。

### 4.2.1 场馆合作管理（平台目录 + 俱乐部挂载）
- **公共场馆目录**（平台后台 / 平台运营）：
  - 维护 `venue_partners` + 平台公共 `venues`（tenant_id=0）；
  - 字段：场馆名 / 联系方式 / 地址 / 营业证照 / 默认营业时段；
  - 审核合作方资质，上下架；状态变更通知挂载该场馆的所有俱乐部。
- **俱乐部合作挂载**（俱乐部后台）：
  - 列表：所有 `club_venue_relations` 记录（自营 / 合作）；
  - 新增：从平台目录搜索 → 挂载 + 选择 `relation_type` + 配置 `allowed_courts`（限定可用场地）+ `settlement_rule`（结算备注 / 抽成规则 / 固定费用）；
  - 编辑 / 暂停 / 解除合作（解除前必须没有未来 14 天内已发布的活动占用，否则强制提示俱乐部先取消活动 / 包场）。
- **冲突告警**：同一场地同一时段被多个俱乐部活动占用时（极端场景：合作方未做独占）后台强制告警 + 拒绝创建活动。

### 4.3 活动管理
- 列表：状态、类型、报名进度（Progress 条）、收入、分享转化（来自 share_token）。
- 创建表单：
  - 类型（散场 / 包场 / 教练课 / 比赛）。
  - 球馆 → 场地块选择（直接选排期块，已被占的不可选）。
  - 价格 / 会员价 / 取消规则（卡片化模板可选）。
  - 报名时间（开始 / 结束）。
- 详情页：
  - Tab1：基本信息（可编辑限定 status<published）。
  - Tab2：报名列表（搜索、备注、取消、改签、手动签到、标记爽约）。
  - Tab3：候补列表（手动递补）。
  - Tab4：财务（收入、退款）。
  - Tab5：分享归因（按 `share_token` 维度展示分享人、渠道、PV、报名转化）。
- 取消活动：弹窗确认 → 全员退款 + 发券（可选）。

### 4.3.1 主理人增长工具（M2 重点）
> 这一组菜单为「让主理人愿意持续发活动」服务，是 SaaS 黏性的关键。

- **活动模板库**：
  - 列表：常用散场 / 包场 / 教练课模板，带使用次数、最近使用时间；
  - 编辑器：固定时间、场馆、人数、价格、取消规则、海报样式、订阅消息模板；
  - 「保存为模板」按钮在活动详情页可一键回填到模板库。
- **批量发布 / 复制活动**：
  - 选模板 → 选日期范围（如未来 4 周每周二、周四 19:00-21:00）→ 批量生成草稿；
  - 一次性确认价格 / 场馆 / 海报后批量发布；
  - 失败项独立列出（如某天场地已被占用），不阻塞其他成功项。
- **周期活动**：固定循环规则（每周 N、每月 N 号）；可暂停 / 跳过单期 / 调整价格。
- **成员导入**：
  - 上传 Excel / CSV → 异步解析 → 进度页（百分比 + 已处理 / 总数 + 实时错误条）；
  - 错误报告下载（行号、字段、原因）；
  - 导入后立即邀请加入俱乐部 + 触发「欢迎首单券」。
  - 关联 API：`POST /members/import` → `GET /members/import/{taskId}`。
- **微信群分享物料**：
  - 海报模板（封面、标题、价格、二维码、活动时间），可主理人自定义文案；
  - 一键导出小程序分享卡片 / 微信群文字 / 朋友圈图片；
  - 每张海报关联 `share_token`，可追踪转化。
- **活动复盘**：
  - 活动结束 24h 自动生成复盘卡片（报名人数、候补人数、爽约率、收入、退款、分享人 Top3）；
  - 一键导出主理人微信，鼓励复盘 + 复用模板。

### 4.4 会员管理
- 列表筛选：等级、标签、消费金额、注册时间、最后活跃、是否黑名单。
- 详情 Drawer：
  - 基础资料 + 会员卡 + 钱包余额 + 报名/订单履历。
  - 操作：调整等级 / 加减积分 / 赠送会员卡 / 余额调整（带审计）/ 拉黑。
- 批量导入：Excel 模板（手机号、姓名、初始等级），后端异步导入 + 进度查询。

### 4.5 钱包 & 财务
- **充值订单**：状态、渠道、金额、是否补退。
- **流水查询**：跨用户聚合查询，支持按 biz_type、时间范围导出。
- **对账中心**：
  - 每日生成对账文件（充值合计、消费合计、退款合计、平台抽成）。
  - 异常告警列表（差错条目）。
- **退款中心**：申请待审核、退款中、已退款；二级审批（>500 主理人审批）。

### 4.6 商城
- 商品 SPU/SKU 管理：
  - 富文本详情（图片粘贴上传 OSS）；
  - 多 SKU 表格编辑（颜色 × 型号矩阵）。
- 订单管理：状态流转、发货（接快递鸟选公司 + 单号、扫描枪粘贴）、自提核销码。
- 退款审核：售后单、聊天记录附件、退款金额改写。
- 库存：单 SKU 流水、批次入库、低库存预警。

### 4.7 教练
- 教练档案 + 课程 + 排期（可视化）+ 学员 + 结算。
- 月度结算单：自动汇总该月已确认课时 → 一键生成 PDF。

### 4.7.1 轻赛事 / 排位赛（M2 起）
- **赛事列表**：状态（草稿 / 已发布 / 进行中 / 已结束 / 已取消）、报名数、比赛进度。
- **赛事创建**：
  - 基本信息：标题、封面、起止日期、绑定球馆、报名费、报名截止；
  - 赛制：MVP 仅支持 `manual` 手动分组 + 手动录入；二期支持 `round_robin` / `knockout` 自动编排；
  - 积分规则（JSON 配置）：胜场积分、败场积分、是否分段位、跨期权重衰减。
- **报名管理**：和活动报名复用结算 / 退款逻辑；可关闭报名 / 手动调整报名状态。
- **比赛录入**：
  - 分组 / 轮次表格视图（Excel-like）；
  - 录入比分时自动计算胜方 + 触发 `ranking_point_logs` 写入；
  - 支持撤销 / 改判（写审计日志）。
- **积分榜 / 段位榜**：按俱乐部维度展示当前积分、近 N 期变化趋势；可按时间段筛选；导出 PDF / Excel。

### 4.7.2 分享归因报表
> 对应 `activity_share_tokens` 表数据。
- **总览**：本周 / 本月 share token 总数、PV、报名转化、报名金额。
- **按分享人**：分享人姓名 → 生成 token 数、PV、报名数、转化率、贡献 GMV，可导出"老带新榜"用于做奖励。
- **按渠道**：`wechat_group / poster / h5 / link` 维度对比，确定哪个渠道最有效。
- **按活动**：单活动维度看哪些活动靠分享带量、哪些靠首页流量。
- **结合 4.3.1 的海报物料**：每张海报、每条群分享都有自己的 `share_token`，可定位到具体一次分享行为。

### 4.8 营销
- 优惠券：列表、新建（满减 / 折扣 / 立减、品类、活动绑定、库存、有效期、领取规则）。
- Banner / 首页运营位：拖拽排序、上下架、定时上线。
- 拼团活动（二期）。
- 分销 / 邀请有礼（二期）。

### 4.9 通知中心
- 后台模板维护（小程序订阅消息模板、站内信模板）。
- 发送日志、失败重试。
- 群发活动（按标签 → 选模板 → 发送 / 定时）。

### 4.10 数据报表
- 报表类型：会员 / 活动 / 财务 / 商品销售 / 教练。
- 通用 ProTable + ECharts 图表混排。
- 导出：异步任务 → 通知中心生成下载链接。

### 4.11 系统
- 员工管理（角色、菜单授权）。
- 角色 / 权限管理（系统角色 + 自定义角色）。
- 操作日志 / 登录日志。
- 系统配置（取消规则、营业默认时段、支付通道开关）。

## 7. 开发与质量

### 7.1 规范
- ESLint + Prettier（标准配置 + AntD 推荐）。
- TS strict mode；禁止 `any`。
- 所有接口必须有 TS 类型（来自 NestJS Swagger 导出的 OpenAPI 生成）。
- SDK 统一使用 `swagger-typescript-api` 生成；禁止手写重复 DTO 类型。
- 业务组件必须有 storybook（关键组件）+ 单测覆盖关键交互。

### 7.2 提交流程
- husky pre-commit：lint-staged + tsc --noEmit。
- commit message：commitlint（Conventional Commits）。
- 分支：`feature/*` / `fix/*` / `release/*`；PR 至 `develop`，发版打 tag 合 `main`。

### 7.3 测试
- **单元测试**：Vitest + Testing Library。
- **接口 mock**：MSW（开发 + 测试通用）。
- **E2E**：Playwright，跑核心黄金路径（登录→建活动→报名→退款）。
- CI：lint + tsc + test + build → 阻止劣化合并。

## 8. 部署

- 静态产物，Nginx / OSS + CDN 托管。
- 多环境：dev / staging / prod，分别走不同 BFF Host（环境变量）。
- 灰度：按租户 ID hash → 新版本 / 旧版本路由（Nginx + cookie）。
